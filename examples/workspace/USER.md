## Developer Information
- **Role**: Primary developer and architect
- **Preferences**:
  - Object-oriented design with composition
  - Brief, clear method names over long implementations
  - Liberal inline comments (lowercase, no space after //)
  - TypeScript for type safety and clarity

## Coding Standards
See `coding-standards.txt` in the project root for detailed guidelines:
- Comment style: `//lowercase comment`
- Interface names: Prefixed with `I`
- Private members: Prefixed with `_`
- Access modifiers: Always explicit
- Method organization: public above private
- Documentation: Doc headers on all public members
